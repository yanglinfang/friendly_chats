### Increment Rules
- **Major bump** (v2 → v3) = 家庭结构/使命出现跨越式变化。  
- **Minor bump** (v3.1, v3.2…) = 当日新增关键事件、技能或系统大修。  
- 文件命名：`adam_backup_v<major>(.<minor>)_YYYYMMDD.md`  
  *示例*：`adam_backup_v3.1_20250425.md`。
